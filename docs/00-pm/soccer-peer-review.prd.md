## 1. PM Discovery — Opportunity Solution Tree

### 1.1 Target Outcome
동네 풋살/축구 동호회원이 경기 후에도 게임을 떠올리며 동호회 활동에 더 깊이 참여한다.

### 1.2 Opportunity Tree

```
[Outcome] 동호회 참여 만족도 & 재참여율 향상
├── [Opp 1] 경기 후 인정받고 싶다 ← 가장 강한 욕구
│   ├── MVP/MOM 투표 결과 즉시 공유
│   ├── 개인 하이라이트 스탯 카드 (SNS 공유용)
│   └── 누적 레이팅 랭킹 보드
│
├── [Opp 2] 내 플레이를 객관적으로 알고 싶다
│   ├── 포지션별 세부 평가 (기술·체력·태도)
│   ├── 시즌별 성장 그래프
│   └── 팀원 코멘트 (익명 피드백)
│
├── [Opp 3] 동호회 운영이 번거롭다
│   ├── 경기 일정·참가자 관리 간소화
│   ├── 카카오톡 공유 링크 한 번으로 리뷰 수집
│   └── 자동 통계 생성 (관리자 작업 제거)
│
└── [Opp 4] 팀 케미가 쌓이길 원한다
    ├── 팀원 간 궁합 지수 (서로 평가 패턴 분석)
    └── 경기별 팀 퍼포먼스 기록
```

### 1.3 Assumption Mapping

| 가정 | 위험도 | 검증 방법 |
|------|--------|-----------|
| 동호회원들이 리뷰 작성에 거부감이 없을 것 | 높음 | MVP 런칭 후 완료율 측정 |
| 익명 리뷰 시 솔직한 평가가 가능할 것 | 중간 | A/B 테스트 (익명 vs 기명) |
| 경기 후 30분 내 리뷰 작성률이 60% 이상일 것 | 높음 | 알림 타이밍 실험 |
| 레이팅 공개가 갈등보다 동기부여를 유발할 것 | 높음 | NPS 조사 |

## 2. PM Strategy — Value Proposition & Lean Canvas

### 2.1 JTBD (Jobs To Be Done) — 6-Part Framework

| Part | 내용 |
|------|------|
| **When** | 풋살 경기가 끝나고 샤워 후 귀가하면서 |
| **I want to** | 오늘 내 플레이가 팀원들에게 어떻게 보였는지 확인하고 싶다 |
| **So I can** | 다음 경기에서 더 잘하고, 팀 내 인정을 받을 수 있다 |
| **But** | 직접 물어보기 어색하고, 단체 카톡에서 피드백을 받기 어렵다 |
| **Which means** | 경기가 끝나면 그냥 잊혀진다 |
| **Help me** | 경기 후 자연스럽게 팀원 평가를 주고받고, 내 성장을 추적할 수 있게 |

### 2.2 Value Proposition Canvas

**고객 프로필 (Customer Profile)**
- Jobs: 경기 참여, 기량 향상, 팀 내 인정, 재미있는 활동
- Pains: 피드백 없음, 경기 기록 소실, 동기부여 저하
- Gains: 인정받고 싶음, 성장 확인, 소속감

**가치 지도 (Value Map)**
- Pain Relievers: 익명 피어리뷰로 솔직한 피드백 수집, 자동 레이팅 집계
- Gain Creators: MVP 투표 공유 카드, 시즌 성장 리포트, 팀 랭킹
- Products: MatchRate 웹 앱

```
  - 경기를 잘해도 팀원들이 알아주는지 모름
  - 내 약점이 뭔지 정확히 피드백받기 어려움
  - 동호회 성적/기록이 카톡에 묻혀버림

  - 본인의 기량이 얼마나 성장했는지 확인
```

## 4. Beachhead Segment (Geoffrey Moore)

### 4.1 Beachhead 정의
**"서울·수도권 기반, 주 1회 이상 정기 경기를 진행하는 20~35인 규모 풋살 동호회 총무/운영자"**

### 4.2 선택 근거
1. **결정권**: 총무가 앱 도입을 결정하면 전체 멤버가 사용
2. **Pain이 명확**: 수동 기록·평가 관리 고통이 즉각적
3. **바이럴 잠재력**: 동호회 간 교류전 → 경쟁 동호회로 자연 확산
4. **측정 가능**: 동호회 수 약 5만 개 (운동조아, 번개장터 스포츠 기준)

### 4.3 Beachhead에서 인접 시장으로
```
[Phase 1] 수도권 풋살 동호회 (MVP)
    ↓ 성공 지표: 동호회 100개, MAU 2,000명
[Phase 2] 전국 풋살 + 11인제 축구 동호회
    ↓ 성공 지표: 동호회 1,000개, MAU 20,000명
[Phase 3] 기업 리그, 대학 동아리, 유소년 클럽
    ↓ 성공 지표: B2B 계약 10건
[Phase 4] 배구, 농구 등 다른 팀 스포츠로 확장
```

## 5. GTM Strategy (Go-To-Market)

### 5.1 론칭 전략

| 단계 | 기간 | 활동 |
|------|------|------|
| **Pre-launch** | D-30 ~ D-1 | 수도권 동호회 총무 50명 베타 초대, 피드백 수집 |
| **Soft Launch** | Week 1~2 | 베타 동호회 10곳 정식 서비스 전환 |
| **Growth** | Month 2~3 | 경기 결과 SNS 공유 카드 → 바이럴 |
| **Scale** | Month 4~6 | 커뮤니티 파트너십 (운동조아, 네이버 카페) |

### 5.2 바이럴 루프 설계
```
경기 후 MVP 발표 → 수상자가 레이팅 카드 SNS 공유
    → 비사용자 클릭 → 랜딩 페이지 → 동호회 등록
    → 팀원 초대 → 리뷰 작성 → 다음 경기 반복
```

### 5.3 채널 전략
1. **카카오톡 공유**: 경기 리뷰 링크를 기존 단체채팅에 공유 (마찰 최소)
2. **인스타그램/X**: 레이팅 카드 자동 생성 → 선수 본인이 공유
3. **네이버 카페**: 풋살 동호회 커뮤니티 직접 홍보
4. **번개스포츠 제휴**: 경기 매칭 후 리뷰 연동

## 6. Product Requirements Document (PRD)

### 6.1 핵심 기능 요구사항

#### FR-01: 동호회 & 경기 관리
- 동호회 생성 및 멤버 초대 (카카오톡 링크)
- 경기 등록 (날짜, 장소, 참가자 선택)
- 팀 구성 (팀 A / 팀 B 자동 배정 또는 수동)
- 경기 결과 입력 (스코어)

#### FR-02: 피어리뷰 시스템
- 경기 종료 후 참가자에게 리뷰 알림 발송
- 리뷰 대상: 같은 경기에 참여한 팀원 + 상대팀 선수
- 평가 항목:
  - **경기력** (1~5점): 기술/드리블/슈팅/패스
  - **체력** (1~5점): 스프린트/지구력/압박
  - **팀플레이 태도** (1~5점): 커뮤니케이션/협동심/스포츠맨십
  - **한 줄 코멘트** (선택, 최대 50자)
- 익명 모드 선택 가능 (동호회 설정)
- 리뷰 제출 후 수정 불가 (24시간 이내)

#### FR-03: 레이팅 집계 & 표시
- 경기별 레이팅 (해당 경기 참가자 평균)
- 누적 레이팅 (전체 경기 가중 평균, 최근 경기 가중치 높음)
- 포지션별 세부 지표 레이더 차트
- 시즌별 성장 그래프

#### FR-04: MVP/MOM 투표
- 경기 참가자 전원이 투표 (자신 제외)
- 투표 결과: MVP (전체), 팀별 MOM 별도 선정
- MVP 발표 카드 생성 (공유용 이미지)

#### FR-05: 개인 프로필 & 성장 리포트
- 선수 카드 (누적 레이팅, 포지션, 경기 수)
- 시즌 리포트 (PDF 다운로드)
- SNS 공유용 스탯 카드 (OG Image 자동 생성)

#### FR-06: 알림
- 경기 등록 시 참가자 카카오 알림
- 경기 종료 후 리뷰 요청 알림 (경기 후 1시간)
- 리뷰 마감 24시간 전 리마인더

### 6.2 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| **성능** | 리뷰 제출 응답시간 < 500ms |
| **모바일 최적화** | iPhone SE 이상 완벽 지원, 터치 타겟 ≥ 44px |

```
Epic 1: 동호회 온보딩
  US-01: 총무가 동호회를 만들고 멤버를 초대할 수 있다
```
