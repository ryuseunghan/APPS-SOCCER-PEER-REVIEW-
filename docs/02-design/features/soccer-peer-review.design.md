## PM × Designer 협의 노트

### 핵심 합의 사항

| 주제 | PM 관점 | Designer 관점 | 합의 결론 |
|------|---------|--------------|-----------|
| 서비스 뎁스 | 리뷰 완료율이 최우선 — 마찰 최소화 | 3 depth 이하로 모든 화면 접근 가능하게 | **최대 3 Depth** 유지, Bottom Nav으로 주요 화면 직접 접근 |
| 리뷰 UX | "30초 리뷰"가 핵심 KPI | 카드 스와이프 or 별점 탭 방식 | **선수별 카드 스와이프** 방식 + 항목은 탭 슬라이더 |
| 색상 톤 | 스포츠 → 활기차고 신뢰감 | 남색 + 초록으로 프리미엄하면서 활동적 | **Deep Navy + Pitch Green** 조합 |
| 정보 밀도 | 통계 데이터가 많음 | 한 화면에 다 보여주면 압도됨 | **Progressive Disclosure** — 핵심 수치 먼저, 탭으로 상세 |
| MVP 공유 카드 | 바이럴 핵심 — 이미지 품질 중요 | 다크 배경 + 그라디언트로 SNS 최적화 | **OG Card** 별도 디자인, 저장 시 고해상도 렌더링 |

### 1.2 Typography

```
Font: Pretendard (한국어 최적화)
Fallback: -apple-system, sans-serif

Scale:
  Display    32px / Bold    → MVP 발표 카드 제목
  H1         24px / Bold    → 페이지 타이틀
  H2         20px / SemiBold→ 섹션 헤딩
  H3         17px / SemiBold→ 카드 제목
  Body       15px / Regular → 일반 텍스트
  Caption    13px / Regular → 보조 정보, 날짜
  Micro      11px / Medium  → 뱃지, 태그
```

### 1.3 Spacing & Radius

```
Spacing: 4px 기준 (4, 8, 12, 16, 20, 24, 32, 48px)
Border Radius:
  sm: 8px   → 태그, 소형 컴포넌트
  md: 12px  → 카드
  lg: 16px  → 모달 시트, 큰 카드
  full: 9999px → 아바타, 뱃지, 버튼 pill

Shadow:
  card: 0 2px 8px rgba(0,0,0,0.12)
  elevated: 0 4px 16px rgba(0,0,0,0.20)
```

### 1.4 Component States

```
Button Primary (Green):
  Default  : bg-green-500  text-white
  Hover    : bg-green-400
  Pressed  : bg-green-600
  Disabled : bg-gray-300  text-gray-500

Star Rating:
  Filled   : text-green-500 (★)
  Half     : text-green-300 (☆)
  Empty    : text-gray-300
```

## 2. Information Architecture (화면 구조)

### 2.1 Navigation Structure (최대 3 Depth)

```
┌─────────────────────────────────────────┐
│              MatchRate App              │
├─────────┬──────────┬────────┬───────────┤
│  홈     │  경기    │ 리뷰   │  내 프로필 │ ← Bottom Tab (L1)
│ (Home)  │ (Match)  │(Review)│ (Profile)  │
└────┬────┴────┬─────┴───┬────┴─────┬─────┘
     │         │         │          │
     │ (L2)    │ (L2)    │ (L2)     │ (L2)
     ▼         ▼         ▼          ▼
  대시보드   경기목록   리뷰 대기   내 스탯카드
  랭킹보드   경기상세   리뷰 작성   동호회 관리
             팀구성/   MVP 투표   성장그래프
             결과입력
                        │ (L3 - 최대)
                        ▼
                    선수별 평가 입력
                    (스와이프 플로우)
```

### 2.2 User Flow — 핵심 경로

```
[신규 사용자]
랜딩 → 카카오 로그인 → 닉네임/포지션 설정 → 동호회 참여(링크) → 홈

[기존 사용자 — 경기 후 리뷰]
푸시 알림 → 리뷰 대기 화면 → 선수 카드 스와이프 → 별점 입력
→ MVP 투표 → 리뷰 완료 → 내 레이팅 공개 → SNS 공유 (선택)

[총무 — 경기 등록]
홈 → [+경기 등록] → 날짜/장소 → 참가자 선택 → 팀 배정 → 결과 입력
```

## 5. Interaction Design

### 5.1 별점 입력 인터랙션

```
1. 탭 즉시 → 별이 채워지며 애니메이션 (scale 1.2 → 1.0)
2. 선택된 별 재탭 → 취소 (별 비워짐)
3. 드래그 → 좌우로 밀면서 별점 조절 가능
4. Haptic → 각 별 선택 시 경미한 진동 (모바일)
```

### 5.2 카드 스와이프 (리뷰 플로우)

```
오른쪽 스와이프 → 다음 선수
왼쪽 스와이프  → 이전 선수
위로 스와이프  → 건너뛰기 (확인 토스트)
아래로 스와이프 → (없음, 스크롤과 충돌 방지)
```

### 5.3 레이팅 공개 애니메이션

```
리뷰 완료 후:
1. 블러 처리된 숫자가 0.5s 페이드인
2. 숫자가 0에서 목표값까지 카운트업 (1.2s)
3. Green glow 이펙트 0.3s (좋은 레이팅 시)
4. 파티클 이펙트 (MVP일 경우 금색 파티클)
```

### 5.4 Loading States

```
스켈레톤 로딩:
  카드 영역 → 펄스 애니메이션 (Navy-800 → Navy-700)
  텍스트 줄  → 회색 바

에러 상태:
  "연결이 끊어졌어요" 토스트 + 재시도 버튼
  오프라인 시 → 캐시된 데이터 표시 (PWA)
```
